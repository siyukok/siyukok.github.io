AndroidVersionIssue

记录android开发中可能遇到的一些问题

  • Android5.0开始使用隐式intent 调用bindService方法,系统会引发异常

  • 使用隐式intent时,若要同时设置 URI 和 MIME 类型，请勿调用 setData() 和 setType()，因为它们会互相抵消彼此的值。请始终使用 setDataAndType() 同时设置 URI 和 MIME 类型。

  • 可能没有任何activity可以接收发出的隐式 Intent。如果出现这种情况，则调用会失败，且应用会崩溃。要验证 Activity 是否会接收 Intent，可以调用intent的resolveActivity()方法。如果结果为非空，则至少有一个应用能够处理该 Intent，且可以安全调用 startActivity()。 如果结果为空，则不应使用该 Intent。

  • 在fragment的onCreate方法中inflater的inflate()方法第三个参数为是否应该在扩展期间将扩展布局附加至 ViewGroup（第二个参数）的布尔值。此处应为 false，因为系统已经将扩展布局插入 container — 传递 true 值会在最终布局中创建一个多余的视图组。
